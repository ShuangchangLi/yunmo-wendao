# 赛博长生：打工修仙录 — 游戏设计文档

> 这份文档是当前迭代版本的 spec，给后续接手的 AI 阅读。
> 游戏老的描述参考 `docs/cyber-xianxia-design.md`（仍保留作为题材线索），但当前 MVP 以本文为准。

## 1. 标题与基调

- **正式标题**：`赛博长生：打工修仙录`
- **副标题 / tagline**：`Cyber Longevity · A Wage-Slave's Cultivation Chronicle`
- **首页文青文案候选**（取一句放在标题下方）：
  - "在 5G 信号最差的天台上，他练第一缕罡气。"
  - "工资发不下来，但师父说，悟道不分早晚。"
  - "天机城的灯不灭，所以这里没有月亮可以养气。"

整体语气：**底层赛博、自嘲、修仙术语外包装**。不要写成"末日逃杀"或"上仙世家"。

## 2. 三页结构（取代原本的单页）

```
[Splash 首页]  -->  [Select 选角 + 选组织]  -->  [Combat 战斗]
   |                                                   |
   +------ <-- [天机档案 Codex]                          v
                                              [Reward 奖励 / Complete / Defeat]
```

### 2.1 Splash 首页

- 全屏赛博城市夜景（沿用 `cyber-city.svg` + `CyberBackground` 涂鸦）。
- 标题大字居中偏上。
- 副标题 + 一句文青 tagline。
- 主菜单按钮（垂直堆叠，居中）：
  - **新的夜巡**（主按钮）
  - **继续存档**（有存档时启用）
  - **天机档案**（codex）
- **不再在首页直接列三个角色**（之前的设计太挤）。
- Web 和 Godot 都使用 `100dvh / 1280×720` 内的固定布局，**不允许出现滚动条**。

### 2.2 选角 + 选组织页（参考 Slay the Spire 2 风）

布局（横屏，1280×720）：

```
+-------------------------------------------------------+
|  [ <- 返回 ]              选择你的人物                |
|                                                       |
|   +-------+   +-------+   +-------+                   |
|   |  老周 |   |  小柯 |   |  阿酥 |  <- 角色卡，竖排 |
|   | 清洁工 |   | 社畜  |   | 主播  |                  |
|   |  立绘 |   |  立绘 |   |  立绘 |                   |
|   |  HP78 |   |  HP84 |   |  HP70 |                   |
|   |  被动 |   |  被动 |   |  被动 |                   |
|   +-------+   +-------+   +-------+                   |
|                                                       |
|              选择你隶属的组织                          |
|                                                       |
|  [天机科技]  [太上学院]  [夜行帮]  <- 组织卡，横排    |
|                                                       |
|                  [ 进入夜巡 ]                          |
+-------------------------------------------------------+
```

- 角色卡和组织卡都点击后高亮（打勾），不立即跳转。
- 底部 `进入夜巡` 按钮，二者都选中后才能点击。
- 角色卡下方写 `名字 · 职业` 两行（不写终末称号、不写组织身份）。

### 2.3 战斗页（StS 风重设计）

布局：

```
+-------------------------------------------------------+
| [退出]  第N战 / 全3战         [档案] [战史]            |  <-- 顶栏（极简）
+-------------------------------------------------------+
|                                                       |
|     +---------+                       +---------+     |
|     |  player |        <对战中央>      |  enemy  |     |  <-- 左：玩家立绘+血条+护体
|     |  立绘   |        （动画位）      |  立绘   |     |       右：敌人立绘+血条+护体+意图
|     |  HP/护  |                       |  HP/护  |     |
|     |  状态   |                       |  意图   |     |
|     +---------+                       +---------+     |
|                                                       |
|  能量                                                 |
|  ●●○                                                  |  <-- 玩家左下角能量
|                                                       |
+-------------------------------------------------------+
| [手牌区]                                  [结束回合]  |
+-------------------------------------------------------+
```

战斗 UI 删除项（**MVP 不显示**）：
- ~~清浊 / 兵性 / 人气~~ 职业资源条
- ~~通玄~~ 状态显示
- ~~身份修行~~ 焦点卡
- ~~抽牌/弃牌堆数显示~~（先去掉，太杂；只在 codex 看，或者后期回归）

战斗 UI 保留项：
- 能量数（当前/上限），显示在玩家立绘的**左下角**。
- HP/护体条，玩家、敌人各一份。
- 敌人意图（攻击/防御/增益 + 数值预告）。
- 灼痕层数（如果有）。
- 手牌、结束回合按钮、战史抽屉。

中间舞台（"动画位"）：
- 暂时画一个空的 panel，留给后期出招动画。
- 现阶段可以是一个赛博风格的"对峙圈"或者一句战斗文案（不是必须）。

## 5. 关键词（精简）

| id | 名字 | 文本 |
| --- | --- | --- |
| `energy` | 能量 | 本回合可用的行动点。每回合开始恢复到 3。 |
| `block` | 护体 | 抵挡即将受到的伤害。玩家护体在自己的回合开始时清空。 |
| `burn` | 灼痕 | 持续伤害。回合开始时受伤，然后层数 -1。 |
| `draw` | 抽牌 | 从牌堆抽指定数量的牌。 |

> **删除**：灵能（重命名为能量）、清浊、兵性、人气、回锋、破网、承压、狂化、弹幕、注目、通玄、身份修行。
> 这些可作为未来扩展回归。

## 7. 战斗规则（极简化）

- 每回合 3 点能量。
- 每回合开始抽 5 张牌。
- 打牌消耗能量。
- 玩家护体在玩家回合开始时清空。
- 敌人护体在敌人回合开始时清空。
- 灼痕在回合开始时造成等量伤害，然后层数 -1。
- 三场战斗后显示"夜巡完成"。

被动技能在每场战斗的中间被自动触发，玩家不感知额外资源条。

## 9. 视觉与排版规范

- Web 端：所有页面用 `height: 100dvh; overflow: hidden`，禁止整页滚动。手牌区可以横向 scroll，但页面级别不允许。
- Godot 端：根 Control 全屏占满，**不要在外层包 ScrollContainer**。所有内容必须在 1280×720 内自适应。
- 像素：所有像素美术 `image-rendering: pixelated` / Godot `TEXTURE_FILTER_NEAREST`。
- 调色板（建议保留）：
  - cyan `#55f7ff` / pink `#ff53d5` / gold `#ffe36f` / red `#ff526f`
  - 深蓝灰底 `#050713`，面板 `rgba(9,13,29,0.86)`

## 10. 数据 schema（两端共用语义）

```ts
type Character = {
  id: "zhou" | "ke" | "su";
  name: string;       // "老周"
  profession: string; // "清洁工"
  tagline: string;    // 一句文青文案
  maxHp: number;
  visual: string;     // pixellab 视觉 id（cleaner / worker / streamer）
  passive: "ji_chen" | "si_xian" | "re_sou";
  deck: string[];     // 起始牌组（cardId 列表）
  rewardPool: string[];
};

type Organization = {
  id: "corp" | "academy" | "gang";
  name: string;
  tagline: string;
  color: string; // hex
};

type Card = {
  id: string;
  name: string;
  cost: number;
  type: "attack" | "skill";
  damage?: number;
  block?: number;
  draw?: number;
  burn?: number;
  heal?: number;
  text: string;
  ownerClass?: "zhou" | "ke" | "su" | "common";
};
```

`PlayerState` 不再包含 `qi / maxQi / transcendent / resourceName`。
新增 `organizationId`，以及被动相关临时计数（`attacksPlayed`、`damagedThisTurn`）。
